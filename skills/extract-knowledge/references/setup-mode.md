# Setup Mode

Interactive configuration. Ask questions one at a time.

## Step 1: Discover Sources

Ask the user:
> "Where are your project directories? (e.g., `~/app`, or comma-separated paths)"

For each path provided:
1. Expand `~` to full home directory path
2. Scan for subdirectories containing `docs/research/`, `docs/`, or `*.md` files
3. For each candidate, count markdown files: `find <path> -name "*.md" | wc -l`
4. Present candidates:
   ```
   Found N projects with research docs:
     financial      → docs/research/ (14 files)
     remotion       → docs/research/ (8 files)
     ...
   Add, remove, or confirm these sources?
   ```
5. Wait for user confirmation. Allow adding/removing entries.

## Step 2: Discover Destinations

Ask the user:
> "Where are your knowledge bases or vaults? (e.g., `~/Obsidian`, or a path)"

For each subdirectory under the provided path:
1. Collect ALL available signals:
   - Check for `.obsidian/` folder → set `type: obsidian-vault`
   - Read `CLAUDE.md` if present → extract description
   - Read `README.md` or `index.md` if present → extract description
   - Scan top-level folder names and sample 3-5 note frontmatter fields → infer purpose
   - If `.obsidian/community-plugins.json` exists → note installed plugins
2. Synthesize signals into a one-line description
3. If no signals found → ask user: "I found `<name>/` but can't determine its purpose. What goes here?"
4. Present each destination:
   ```
   ai_explosion_kb — detected from CLAUDE.md:
     "AI/ML, frontend, career development, learning, side projects"
   Is this accurate? (yes / edit)
   ```
5. After all destinations confirmed, ask:
   > "Which is the default destination? (used when routing is uncertain)"

## Step 3: Write Config

Write `<project>/.claude/extract-knowledge.yaml` with the confirmed sources and destinations.

```yaml
# Generated by /m-extract-knowledge setup on <date>

sources:
  - path: <expanded full path>
    label: <directory name>
  # ... one per source

destinations:
  - path: <expanded full path>
    label: <directory name>
    description: "<detected or user-provided description>"
    detected_from: <CLAUDE.md | README.md | index.md | folder_scan | user_input>
    type: <obsidian-vault | plain-markdown>
    default: true  # only on the user-selected default
  # ... one per destination

reports_dir: .claude/extract-knowledge/reports
```

Create the reports directory:
```bash
mkdir -p <project>/.claude/extract-knowledge/reports
```

Print: "Config saved. Run `/m-extract-knowledge` to start extraction."
